# Desarrollo frontend

## HTML

Uso estructura semántica, encabezados jerárquicos, textos alternativos, etiquetas de formulario y botones reales para acciones.

## CSS

Mantengo variables de diseño, componentes reutilizables y puntos de quiebre razonables. Los efectos visuales no deben alterar el tamaño del contenido ni bloquear la interacción.

## JavaScript

Cada archivo tiene una responsabilidad. Valido elementos antes de usarlos, escucho eventos de forma delegada cuando el contenido es dinámico y evito temporizadores innecesarios.

## Interacciones Etapa 35

Los CTA principales reciben elevación, brillo y respuesta de presión. Las acciones secundarias usan efectos más discretos. Los filtros tienen estados activos claros. “Agregar a mi selección” muestra confirmación temporal. Todo respeta `prefers-reduced-motion`.
