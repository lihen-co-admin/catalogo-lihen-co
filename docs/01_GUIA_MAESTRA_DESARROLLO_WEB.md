# Guía maestra para desarrollar un aplicativo web

## Ruta general

**Idea → análisis → requerimientos → diseño → prototipo → desarrollo → pruebas → publicación → mantenimiento.**

## 1. Idea y problema

Yo comienzo describiendo la necesidad, las personas afectadas y el cambio que se espera lograr. Luego defino objetivo general, objetivos específicos, alcance, limitaciones y criterios de éxito.

## 2. Requerimientos

Separo requerimientos funcionales —lo que el sistema hace— de no funcionales —seguridad, rendimiento, accesibilidad, compatibilidad y facilidad de uso—. Cada historia de usuario debe tener criterios de aceptación comprobables y prioridad.

## 3. Diseño previo

Antes de programar preparo identidad visual, moodboard, paleta, tipografías, arquitectura de información, mapa de navegación, flujo de usuario, wireframes, mockups y prototipo. El diseño debe contemplar estados normales, vacíos, errores y confirmaciones.

## 4. Construcción

Organizo carpetas y asigno una responsabilidad clara a cada archivo. Construyo primero la estructura HTML, después la presentación CSS y finalmente la interacción JavaScript. Cuando exista backend, documento API, autenticación, roles, base de datos y manejo de errores.

## 5. Responsive y accesibilidad

Compruebo celular, tableta, portátil, escritorio y pantallas grandes. Los botones deben ser táctiles, el texto legible, las imágenes flexibles y la navegación utilizable con teclado. Televisor, proyector y Smart TV se analizan como contextos diferentes.

## 6. Pruebas

Pruebo cada módulo, los formularios con datos válidos e inválidos, navegación, enlaces, rendimiento, compatibilidad y experiencia real de usuario. Registro el resultado esperado, el obtenido, la evidencia y la corrección.

## 7. Publicación

Preparo producción, reviso secretos, ejecuto validaciones, creo commit, hago push, verifico GitHub Pages o Netlify y pruebo el enlace público desde varios dispositivos.

## 8. Documentación y mantenimiento

Conservo manual técnico, manual de usuario, decisiones, versiones, incidencias, respaldo y lista priorizada de mejoras. Cada actualización debe poder entenderse y comprobarse.

## Distinción de fuentes

En una entrega académica separo claramente: contenido enseñado en clase, aplicación al proyecto, temas pendientes y recomendaciones profesionales adicionales.
